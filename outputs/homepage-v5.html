What is a product graph?

The structured source of truth behind scalable, valid environment generation — geometry, dimensions, materials, rules, articulation, semantics, and spatial constraints.

scene_graph.json 
KitchenEnvironment {
  Cabinet_01: { label: "upper_cabinet", id: "0x001" }
    ├─ Door_Left  (hinge_joint, 0°–110°)
    └─ Door_Right (hinge_joint, 0°–110°)
  Refrigerator: { label: "refrigerator", id: "0x004" }
    └─ Door       (articulated_joint, 0°–90°)
  Dishwasher:   { label: "dishwasher",   id: "0x007" }
    └─ Drawer     (prismatic_joint, 0–520mm)
  Countertop:   { label: "countertop", material: "granite_01" }
}
// Labels like dishwasher, base_cabinet, countertop
// — generated at creation, not post-processed

How It Works

The system that produced what you just saw.

Product Models → Product Graph → Environment Generator → Scene Graph → USD Export → Simulation Platforms. Every stage is structured, validated, and reproducible.

01

Fully tagged scene graph — generated at creation

Every object carries its full identity from creation: semantic class, instance ID, dimensions, spatial transform. Not post-processed annotation — generated at creation from the product graph, because the graph defines what everything is. Labels like dishwasher, base_cabinet, countertop — with instance IDs and parent-child hierarchies — are structural outputs, not afterthoughts.

02

Parameter-driven variation at dataset scale

Materials, lighting, layout permutation, object placement — all parameter-driven. Every variant is structurally valid. Robotics teams use this for distribution coverage: same scene, different materials, different lighting, different clutter states. 1 base environment × 9 cabinet finishes × 4 countertop materials × 4 lighting conditions × 3 layout types = 432 unique valid environments — plus clutter and spatial randomisation → tens of thousands of training scenes. Available via API.

03

Perception training on physically accurate environments

Every surface has physically accurate PBR materials derived from structured product data. Every layout is dimensionally accurate and physically plausible. Every object is semantically labeled. Your models train on grounded environments — not procedural approximations or AI-synthesized scenes.
