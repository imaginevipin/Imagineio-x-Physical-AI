Developer API

Integrate in minutes.
Train at scale.

Query any environment, generate thousands of variations, and export directly to your simulator — all from a single Python call.

train_pipeline.py 
import imagineo

# Initialise client
client = imagineo.Client(api_key="YOUR_API_KEY")

# Query sim-ready environments
envs = client.environments.query(
    category="kitchen",
    sim_ready=True,
    export_format="usd"
)

# Generate 10,000× variations
batch = client.generate(
    environment_id=envs[0].id,
    count=10_000,
    params={
        "lighting": "randomized",
        "layout":   "parametric",
        "materials": "varied",
    }
)

# Export to NVIDIA Isaac Sim
client.export(batch, target="isaac_sim")
# ✓ Ready in < 5 minutes

Solution

One platform. Every step of the pipeline.

Ingest → normalise → augment → validate → deploy. End-to-end data infrastructure, purpose-built for Physical AI training workflows.

01

Ingest

Structured product data enters via API, batch upload, or direct integration with your existing catalogue systems. Proprietary CAD, GLTF, FBX — all accepted.

02

Normalise

Every asset is normalised to a unified product graph — geometry, semantics, physics, and 150+ metadata attributes. Consistent, structured, reproducible.

03

Augment

Parametric variation engine generates up to 10,000× variants per environment across lighting, material, layout, and scene composition.

04

Validate

Every output passes physics integrity checks, collision validation, and semantic labelling verification before it enters your pipeline. Zero manual QA.

05

Deploy

Export to Isaac Sim, MuJoCo, Unreal, or OpenUSD. Load time under 5 minutes. No manual scene setup. No format conversion. Just data.
