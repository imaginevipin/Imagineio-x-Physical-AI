# Outputs

All project deliverables live here. Each piece of work gets its own folder, and each iteration is a separate variation inside it.

## Structure

```
outputs/
└── [work-name]/
    ├── v1/       ← first version
    ├── v2/       ← second iteration
    └── v3/       ← and so on
```

## Naming Convention

- Folder names: lowercase, hyphenated (e.g. `landing-page`, `hero-section`, `pricing-table`)
- Variations: `v1`, `v2`, `v3` — never overwrite a previous version, always create a new one
- Each variation is self-contained (all its files live inside its own folder)
